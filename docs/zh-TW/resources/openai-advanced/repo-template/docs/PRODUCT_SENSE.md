## 產品規則

- 優先保證使用者可感知的可靠性，而不是一味加功能。
- 行為有歧義時，把它當成 spec 缺口，不要預設允許猜。
- 只要實現改變了使用者看見或信任的東西，就更新對應 spec。
- 具體流程寫在 product spec 裡，跨流程的產品優先級寫在這裡。

## 禁區模式

- 隱蔽的破壞性操作
- 沒有使用者回饋的靜默失敗
- 使用者可見狀態沒有明確真相來源
- 不能用一句話解釋清楚的功能
